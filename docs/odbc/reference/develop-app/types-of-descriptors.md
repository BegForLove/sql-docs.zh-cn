# <a name="types-of-descriptors"></a>描述符类型
描述符用于描述以下项之一：  
  
-   一组零个或多个参数。 可以使用参数描述符来描述：  
  
    -   *应用程序参数缓冲区*其中包含作为组由应用程序或执行后输出动态参数的输入动态参数**调用**的 SQL 语句。  
  
    -   *实现参数缓冲区*。 对于输入的动态参数，此应用程序可以指定任何数据转换后包含与应用程序参数缓冲区相同的参数。 对于输出的动态参数，其中包含返回自变量，该应用程序可以指定任何数据转换前。  
  
     对于输入的动态参数，该应用程序必须对应用程序参数描述符之前执行任何 SQL 语句包含动态参数标记。 对于输入和输出的动态参数，该应用程序可以指定不同的数据类型与实现参数描述符中来实现数据转换。  
  
-   单个数据库数据行。 可以使用行描述符来描述：  
  
    -   *实现行缓冲区*其中包含从数据库的行。 （这些缓冲区从概念上讲写入到包含数据或从数据库读取。 但是，未指定数据库数据的存储的形式。 数据库可以执行其他转换上从其窗体实现缓冲区中的数据。）  
  
    -   *应用程序行缓冲区*呈现给应用程序，该应用程序可以指定任何数据转换之后，其中包含的数据行。  
  
     应用程序对操作的应用程序行描述符在任何情况下数据库中的列数据应用程序变量中必须出现的位置。 若要实现列数据的数据转换，该应用程序可以指定不同的数据类型中实现行描述符。  
  
 下表总结了描述符类型。  
  
|缓冲区类型|“行”|动态参数|  
|-----------------|----------|------------------------|  
|**应用程序缓冲区**|应用程序行描述符 (ARD)|应用程序参数描述符 (APD)|  
|**实现缓冲区**|实现行描述符 (IRD)|实现参数描述符 (IPD)|  
  
 为参数或行缓冲区时，如果应用程序的实现和应用程序描述符的相应记录中指定不同的数据类型驱动程序执行数据转换时使用描述符。 例如，它可能会将数字和日期时间值转换为字符串格式。 (有关有效转换，请参阅[附录 d： 数据类型](../../../odbc/reference/appendixes/appendix-d-data-types.md)。)  
  
 描述符可以执行不同的角色。 不同的语句可以共享任何应用程序显式分配的描述符。 在一个语句中的行描述符可以作为另一个语句中的参数描述符。  
  
 它始终为已知给定的描述符是应用程序描述符或实现描述符，即使描述符具有尚未使用在数据库操作。 为实现隐式分配的描述符，实现记录的预定义的行相对于语句句柄。 应用程序将通过调用其分配任何描述符**SQLAllocHandle**不是应用程序描述符。
