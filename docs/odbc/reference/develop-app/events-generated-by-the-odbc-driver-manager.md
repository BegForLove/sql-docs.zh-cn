# <a name="events-generated-by-the-odbc-driver-manager"></a>生成的 ODBC 驱动程序管理器事件
> [!IMPORTANT]  
>  Windows 8 （Visual Studio Analyzer 仅包含在较旧版本的 Visual Studio。） 从开始已删除的 Visual Studio Analyzer 的支持。 有关故障排除机制的替代方法，使用 BID 跟踪。  
  
 单击启动 Visual Studio Analyzer 按钮时注册所生成的 ODBC 驱动程序管理器的事件。 本身的工具提供了系统定义的事件和创建自定义事件的能力。 有关事件的详细信息，请参阅*Visual Studio Analyzer 参考指南*文档的 Visual Studio 套件中。  
  
|Visual Studio 分析器事件|Description|  
|----------------------------------|-----------------|  
|**调用**|在每个 ODBC API 条目时生成。|  
|**ReturnException**|如果返回的代码是 SQL_ERROR，生成每个 ODBC API 返回。|  
|**ReturnNormal**|如果返回的代码不是 SQL_ERROR，生成每个 ODBC API 返回。|  
|**连接开始**|指示连接启动;ODBC 驱动程序管理器调用驱动程序的连接 Api 时，生成。|  
|**完成连接**|指示连接完成;驱动程序的连接 Api 返回到 ODBC 驱动程序管理器时，生成。|  
|**断开连接启动**|ODBC 驱动程序管理器调用的驱动程序时生成**SQLDisconnect**函数。|  
|**断开连接完成**|生成时驱动程序的**SQLDisconnect**函数将返回到 ODBC 驱动程序管理器。|  
|**QuerySend**|ODBC 驱动程序管理器调用的驱动程序时生成**SQLPrepare**， **SQLExecute**， **SQLExecDirect**函数，以及目录函数如**SQLTables**和**SQLColumns**。|  
|**QueryResult**|该驱动程序返回的结果集到 ODBC 驱动程序管理器涉及查询的函数时生成。|  
|**TransactionStart**|生成应用程序将 SQL_ATTR_AUTOCOMMIT 的值设置为 SQL_AUTOCOMMIT_OFF，或者在应用程序已成功调用后**SQLEndTran**。|  
|**TransactionCommit**|当应用程序调用时，生成**SQLEndTran**提交本地事务。|  
|**TransactionRollback**|当应用程序调用时，生成**SQLEndTran**回滚本地事务。|  
|**JoinDTC**|当应用程序加入分布式事务处理协调器 (DTC) 生成。|  
|**LeaveDTC**|生成应用程序退出分布式事务处理协调器 (DTC) 时。|
