# <a name="types-of-descriptors"></a>描述符类型
描述符用于描述以下描述之一：  
  
-   一组零个或多个参数。 参数描述符可用于描述：  
  
    -   *应用程序参数缓冲区，* 它包含应用程序设置的输入动态参数或 SQL 的**CALL**语句执行后的输出动态参数。  
  
    -   *实现参数缓冲区*。 对于输入动态参数，它包含与应用程序参数缓冲区相同的参数，在应用程序可以指定的任何数据转换之后。 对于输出动态参数，这包含返回的参数，在应用程序可以指定的任何数据转换之前。  
  
     对于输入动态参数，应用程序在执行包含动态参数标记的任何 SQL 语句之前，必须对应用程序参数描述符进行操作。 对于输入和输出动态参数，应用程序可以指定与实现参数描述符中的数据类型不同的数据类型，以实现数据转换。  
  
-   一行数据库数据。 行描述符可用于描述：  
  
    -   *实现行缓冲区，* 其中包含数据库中的行。 （这些缓冲区在概念上包含写入数据库或从数据库读取的数据。 但是，未指定数据库数据的存储形式。 数据库可以在实现缓冲区中对其形式中的数据执行其他转换。  
  
    -   *应用程序行缓冲区，* 其中包含向应用程序显示的数据行，在应用程序可能指定的任何数据转换之后。  
  
     应用程序在应用程序行描述符上运行，在任何情况下，数据库中的列数据必须出现在应用程序变量中。 为了实现列数据的数据转换，应用程序可以指定与实现行描述符中的数据类型不同的数据类型。  
  
 描述符类型在下表中汇总。  
  
|缓冲区类型|“行”|动态参数|  
|-----------------|----------|------------------------|  
|**应用程序缓冲区**|应用程序行描述符 （ARD）|应用程序参数描述符 （APD）|  
|**实现缓冲区**|实现行描述符 （IRD）|实现参数描述符 （IPD）|  
  
 对于参数或行缓冲区，如果应用程序在实现和应用程序描述符的相应记录中指定不同的数据类型，则驱动程序在使用描述符时执行数据转换。 例如，它可以将数值和日期时间值转换为字符串格式。 （有关有效的转换，请参阅[附录 D：数据类型](../../../odbc/reference/appendixes/appendix-d-data-types.md).）  
  
 描述符可以执行不同的角色。 不同的语句可以共享应用程序显式分配的任何描述符。 一个语句中的行描述符可以用作另一个语句中的参数描述符。  
  
 始终知道给定描述符是应用程序描述符还是实现描述符，即使描述符尚未在数据库操作中使用也是如此。 对于实现隐式分配的描述符，实现记录相对于语句句柄的预定义行。 应用程序通过调用**SQLAllocHandle**分配的任何描述符都是应用程序描述符。
