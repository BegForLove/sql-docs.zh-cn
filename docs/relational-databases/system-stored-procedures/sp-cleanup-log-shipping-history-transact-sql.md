# <a name="spcleanuplogshippinghistory-transact-sql"></a>sp_cleanup_log_shipping_history (Transact-SQL)
[!INCLUDE[tsql-appliesto-ss2008-xxxx-xxxx-xxx-md](../../includes/tsql-appliesto-ss2008-xxxx-xxxx-xxx-md.md)]

  此存储过程将根据保持期，清理本地和监视服务器上的历史记录。  
  
 ![主题链接图标](../../database-engine/configure-windows/media/topic-link.gif "主题链接图标") [TRANSACT-SQL 语法约定](../../t-sql/language-elements/transact-sql-syntax-conventions-transact-sql.md)  
  
## <a name="syntax"></a>语法  
  
```  
  
sp_cleanup_log_shipping_history  
[ @agent_id = ] 'agent_id',  
[ @agent_type = ] 'agent_type'  
```  
  
## <a name="arguments"></a>参数  
`[ @agent_id = ] 'agent_id',` 用于备份的主 ID 或者用于复制或还原的辅助 ID。 *agent_id*是**uniqueidentifier**且不能为 NULL。  
  
`[ @agent_type = ] 'agent_type'` 日志传送作业的类型。 0 = 备份，1 = 复制，2 = 还原。 *agent_type*是**tinyint**且不能为 NULL。  
  
## <a name="return-code-values"></a>返回代码值  
 0（成功）或 1（失败）  
  
## <a name="result-sets"></a>结果集  
 无。  
  
## <a name="remarks"></a>备注  
 **sp_cleanup_log_shipping_history**必须从运行**主**任何日志传送服务器上的数据库。 此存储的过程清理的本地和远程副本**log_shipping_monitor_history_detail**并**log_shipping_monitor_error_detail**根据历史记录保持期。  
  
## <a name="permissions"></a>权限  
 只有的成员**sysadmin**固定的服务器角色可以运行此过程。
