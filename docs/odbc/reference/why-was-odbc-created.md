# <a name="why-was-odbc-created"></a>为何创建 ODBC？
从历史上看，公司使用单个 DBMS。 通过该系统的前端或通过编写以独占方式使用该系统的应用程序，已完成的所有数据库访问。 但是，在使用计算机的大小增长和更多计算机硬件和软件变得可用时，公司开始获取不同的 Dbms。 之所以很多原因：人们购买了什么是成本最低，什么是最快，他们已经知道，是什么最适合单个应用程序在市场上最新。 其他原因是重组和合并，其中部门以前单一 DBMS 现在必须在多个。  
  
 此问题的大小增长随着的个人计算机变得更加复杂。 这些工具，用于查询、 分析和显示数据，以及数量的廉价、 易于使用的数据库的主机中的计算机。 此后，可以在单个企业通常具有数据分散在大量的台式计算机、 服务器和小型计算机、 存储在多个不兼容的数据库，并由大量不同的工具，其中很少有人能够到达的所有数据的访问。  
  
 最后一个质询总结了出现客户端/服务器计算，希望找到可以最有效利用计算机资源。 成本较低的个人计算机 （客户端） 位于在桌面上，提供这两个图形前端的数据的成本较低的工具，如电子表格、 图表程序和报表生成人员数。 小型计算机和大型计算机 （服务器） 托管的 Dbms，其中他们可以使用其的计算能力和中心位置来提供快速、 协调数据的访问权限。 如何然后是要连接到后端数据库的前端软件？  
  
 类似的问题所面临独立软件供应商 (Isv)。 编写数据库软件，用于小型计算机和大型机供应商通常必须为每个 DBMS 编写的应用程序的一个版本或为他们想要访问每个 DBMS 编写特定于 DBMS 的代码。 编写为个人计算机的软件供应商必须为每个需要处理的不同 DBMS 编写数据访问例程。 这通常意味着编写和维护数据访问例程，而不是应用程序，以及应用程序通常出售未对其质量，但它们无法访问给定 DBMS 中的数据是否在花费大量的资源。  
  
 这两个集的开发人员所需要的是不同的 Dbms 中访问数据的方式。 大型机和小型计算机组需要通过某种方式将数据从单个应用程序中的不同 Dbms 合并时的个人计算机组需要此功能，以及编写了独立于任何一个 DBMS 的单个应用程序的方式。 简单地说，这两个组需要访问数据; 的可互操作方法它们需要开放式数据库连接。
