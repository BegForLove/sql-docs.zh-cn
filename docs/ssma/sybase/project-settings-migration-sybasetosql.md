# <a name="project-settings-migration-sybasetosql"></a>项目设置（迁移）(SybaseToSQL)
迁移页**项目设置**对话框中包含自定义如何 SSMA 迁移数据从 Sybase Adaptive Server Enterprise (ASE) 中，为设置[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]。  
  
迁移窗格中现已推出两**项目设置**并**默认项目设置**对话框。  
  
-   若要指定所有的 SSMA 项目的设置**工具**菜单中，选择**默认项目设置**，选择迁移项目类型设置为其所需查看 / 更改从**迁移目标版本**下拉列表单击**常规**底部的左窗格中，然后单击**迁移**。  
  
-   若要在指定的当前项目中，设置**工具**菜单中，选择**项目设置**，单击**常规**底部的左窗格中，然后单击**迁移**。  
  
## <a name="date-correction-options"></a>日期更正选项  
  
|术语|定义|  
|--------|--------------|  
|**替换为不受支持的日期**|指定是否 SSMA 应该更正日期早于最早[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] **datetime**日期 (01 1753 年 1 月)。<br /><br />若要保留当前的日期值，选择**不执行任何操作**。 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 日期时间列中将不接受 01 1753 年 1 月之前的日期。 如果使用较旧的日期，必须将日期时间值转换为字符值。<br /><br />若要将 01 1753 年 1 月之前的日期转换为 NULL，请选择**替换为 NULL**。<br /><br />若要将受支持的日期替换为 01 1753 年 1 月之前的日期，请选择**替换为最近的受支持的日期**。<br /><br />**默认模式**:不执行任何操作<br /><br />**乐观模式**:不执行任何操作<br /><br />**完整模式**:替换为最近的受支持的日期|  
  
## <a name="migration-engine"></a>迁移引擎  
  
|术语|定义|  
|--------|--------------|  
|**迁移引擎**|指定在数据迁移过程中使用的数据库引擎。 客户端数据迁移是指从源和大容量将该数据插入到 SQL Server 中检索数据的 SSMA 客户端。 服务器端数据迁移是指 SSMA 数据迁移引擎 （大容量复制程序） 框中运行 SQL Server 作为 SQL 代理作业从源检索数据并将直接插入 SQL Server，从而避免额外客户端跃点 （更好的性能）。<br /><br />**默认模式**:客户端数据迁移引擎<br /><br />**乐观模式**:客户端数据迁移引擎<br /><br />**完整模式**:客户端数据迁移引擎|  
  
> [!IMPORTANT]  
> 当**迁移引擎**选项设置为**服务器端数据迁移引擎**，将选项设置的新项目**使用 32 位服务器端数据迁移引擎**显示. 它指定是否使用 32 位或 64 位的大容量复制程序 (BCP) 实用程序来迁移数据。  
  
## <a name="miscellaneous-options"></a>其他选项  
  
|术语|定义|  
|--------|--------------|  
|**批大小**|指定批处理大小在数据迁移过程中使用。<br /><br />**默认模式**:10000<br /><br />**乐观模式**:10000<br /><br />**完整模式**:10000|  
|**检查约束**|指定是否将数据插入到 SQL Server 表时，SSMA 应检查约束。<br /><br />**默认模式**:False<br /><br />**乐观模式**:False<br /><br />**完整模式**:False|  
|**数据迁移超时**|指定在数据迁移过程中使用的超时<br /><br />**默认模式**:15<br /><br />**乐观模式**:15<br /><br />**完整模式**:15|  
|**扩展的数据迁移选项**|在单独的详细信息选项卡中显示每个表的额外的数据迁移选项。<br /><br />**默认模式**:隐藏<br /><br />**乐观模式**:隐藏<br /><br />**完整模式**:隐藏|  
|**激发触发器**|指定当其将数据添加到 SQL Server 表 SSMA 是否应激发插入触发器。<br /><br />**默认模式**:False<br /><br />**乐观模式**:False<br /><br />**完整模式**:False|  
|**保留标识**|指定时它将数据添加到 SQL Server，SSMA 是否保留 Sybase 标识值。 值为 False 将导致由目标分配标识值。<br /><br />**默认模式**:True<br /><br />**乐观模式**:True<br /><br />**完整模式**:True|  
|**保留 Null**|指定是否 SSMA 保留源数据中的 null 值时它将数据添加到 SQL Server 的而不考虑 SQL Server 中指定的默认值。<br /><br />**默认模式**:True<br /><br />**乐观模式**:True<br /><br />**完整模式**:True|  
|**出错时**|发生错误时停止数据迁移。 它具有三个选项：<br /><br />**停止迁移：** 停止数据迁移操作<br /><br />**转到下一个表：** 停止数据迁移到当前表和继续到下一个<br /><br />**请继续阅读下一批：** 停止数据迁移到当前批次，并将继续到下一个<br /><br />**默认模式**:请继续执行下一批<br /><br />**乐观模式**:请继续执行下一批<br /><br />**完整模式**:请继续执行下一批|  
|**数字舍入小数部分。**|指定是否为整数类型的迁移过程修整 decimal 和 numeric 数据的小数部分或显示错误消息，如果小数部分很重要<br /><br />**默认模式**:否<br /><br />**乐观模式**:否<br /><br />**完整模式**:否|  
|**Sybase Unicode 字节序**|指定的字节序 Sybase Unicode 字符串类型。 为此特定设置，可以设置以下选项：<br /><br />小字节序<br /><br />Big endian<br /><br />**默认模式**:小字节序<br /><br />**乐观模式**:小字节序<br /><br />**完整模式**:小字节序|  
|**表锁**|指定时它将数据添加到表数据迁移期间，SSMA 是否锁定表。 大容量复制操作期间获取大容量更新锁。 如果值为 False，在行级别设置锁。<br /><br />**默认模式**:True<br /><br />**乐观模式**:True<br /><br />**完整模式**:True|  
|**使用游标**|如果设置此选项使用游标的源数据库中检索的数据。<br /><br />**默认模式**:False<br /><br />**乐观模式**:False<br /><br />**完整模式**:False|  
  
## <a name="parallel-data-migration"></a>并行数据迁移  
  
|术语|定义|  
|--------|--------------|  
|**并行数据迁移模式下**|指定到分叉线程用来启用并行数据迁移的模式。 在自动模式下，SSMA 选择的线程数 (默认值为 10) 将数据迁移进行分叉。 在自定义模式下，用户可以指定要将数据迁移进行分叉的线程数 （最小值为 1，最大值为 100）。 目前，仅客户端数据迁移引擎支持并行数据迁移。<br /><br />**默认模式**:Auto<br /><br />**乐观模式**:Auto<br /><br />**完整模式**:Auto|  
  
> [!IMPORTANT]  
> 当**并行数据迁移模式下**选项设置为**自定义**，将选项设置的新项目**线程计数**显示。 它指定用于满足数据迁移的线程数。
