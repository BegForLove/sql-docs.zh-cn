# <a name="odbc-glossary"></a>ODBC 术语表
## <a name="a"></a>A  
 **访问计划**  
 数据库引擎生成的执行计划的 SQL 语句。 等效于从 C.等第三代语言编译的可执行代码  
  
 **聚合函数**  
 从一组值，通常用于生成单个值的函数**GROUP BY**并**HAVING**子句。 聚合函数包括**AVG**，**计数**，**最大**， **MIN**，并**总和**。 也称为*集函数*。 *另请参阅*标量函数。  
  
 **ANSI**  
 美国国家标准学会。 ODBC API 基于 ANSI 调用级别接口。  
  
 **APD**  
 *请参阅*应用程序参数描述符 (APD)。  
  
 **API**  
 应用程序编程接口。 一组应用程序使用请求和较低级别服务执行的例程。 ODBC API 组成的 ODBC 函数。  
  
 **application**  
 ODBC API 中调用函数的可执行程序。  
  
 **应用程序参数描述符 (APD)**  
 介绍在应用程序指定任何转换之前的 SQL 语句中使用的动态参数描述符。  
  
 **应用程序行描述符 (ARD)**  
 表示的列元数据和应用程序的缓冲区，描述以下应用程序指定任何数据转换的数据行中的数据描述符。  
  
 **ARD**  
 *请参阅*应用程序行描述符 (ARD)。  
  
 **自动提交模式**  
 在其中事务后立即被提交也会执行这些事务提交模式。  
  
## <a name="b"></a>B  
 **行为更改**  
 在 ODBC 中的某些功能的更改*3.x*到 ODBC 行为*2.x*行为，反之亦然。 引起更改 SQL_ATTR_ODBC_VERSION 环境属性。  
  
 **二进制大型对象 (BLOB)**  
 超过某个数目的字节，如 255 任何二进制数据。 通常更长的时间。 此类数据通常是发送到和从部件中的数据源中检索。 也称为*长整型数据*。  
  
 **binding**  
 为谓词，将在结果集中的列或参数中的 SQL 语句与应用程序变量相关联的操作。 为名词，关联。  
  
 **绑定偏移量**  
 有关所有绑定列或参数的数据，生成新地址添加到数据缓冲区地址和长度/指示器缓冲区地址的值。  
  
 **块游标**  
 游标支持的一次提取多行数据。  
  
 **缓冲区**  
 一种用于将应用程序和驱动程序之间传递数据的应用程序内存。 缓冲区通常成对出现：*数据缓冲区*和一个*数据长度缓冲区*。  
  
 **byte**  
 8 位或一个八位字节。 *另请参阅*八位位组。  
  
## <a name="c"></a>C  
 **C 数据类型**  
 在 C 程序中，在这种情况下应用程序中的变量数据类型。  
  
 **catalog**  
 系统数据库的表中描述的数据库的形状的集。 也称为*架构*或*数据字典*。  
  
 **目录函数**  
 ODBC 函数，用于从数据库的目录中检索信息。  
  
 **CLI**  
 *请参阅*API。  
  
 **client/server**  
 在其中一个或多个客户端数据通过服务器访问该数据库的访问策略。 客户端通常实现用户界面，而服务器控制数据库访问权限。  
  
 **column**  
 用于单个行中的信息项的容器。 也称为*字段*。  
  
 **commit**  
 若要永久所做的更改在事务中。  
  
 并发   
 若要在同一时间访问相同的数据的多个事务能力。  
  
 **一致性级别**  
 一组离散的驱动程序或数据源支持的功能。 ODBC 定义 API 一致性级别和 SQL 一致性级别。  
  
 连接   
 驱动程序和数据源的特定实例。  
  
 **连接浏览**  
 搜索数据源连接到网络。 连接浏览可能涉及到几个步骤。 例如，用户可能会首先浏览网络以查找服务器，然后浏览特定数据库的服务器。  
  
 **连接句柄**  
 包含有关连接的信息的数据结构句柄。  
  
 **当前行**  
 游标当前指向的行。 定位的操作将作用于当前行。  
  
 **cursor**  
 一种返回应用程序的数据行的软件。 终端; 的计算机上闪烁的光标位置后可能名为就像该光标指示屏幕上的当前位置，结果集的游标指示结果集中当前位置。  
  
## <a name="d"></a>D  
 **数据缓冲区**  
 用于将数据传递的缓冲区。 通常关联的数据缓冲区已*数据长度缓冲区*。  
  
 **数据字典**  
 *请参阅*目录。  
  
 **数据长度的缓冲区**  
 用于传递中相应的值的长度的缓冲区*数据缓冲区*。 数据长度缓冲区还用于存储指标，例如数据值是否是以 null 结尾。  
  
 **数据源**  
 （如果有），用户想要访问其关联的操作系统、 DBMS 和网络平台的数据。  
  
 **数据类型**  
 一段数据的类型。 ODBC 定义 C 和 SQL 数据类型。 *另请参阅*类型指示符。  
  
 **data-at-execution column**  
 为其发送数据后的列**SQLSetPos**调用。 如此命名是因为在执行时间而不是行集缓冲区中放入发送数据。 在执行时的部分通常发送长数据。  
  
 **执行时数据参数**  
 为其数据发送后的参数**SQLExecute**或**SQLExecDirect**调用。 如此命名是因为当 SQL 语句执行而不是参数缓冲区中放入发送数据。 在执行时的部分通常发送长数据。  
  
 **database**  
 离散 DBMS 中数据的集合。 此外 DBMS。  
  
 **数据库引擎**  
 分析和执行 SQL 语句并访问物理数据的 DBMS 中的软件。  
  
 **DBMS**  
 数据库管理系统。 物理数据库和用户之间的软件层。 该 DBMS 管理对数据库的所有访问。  
  
 **基于 DBMS 的驱动程序**  
 通过独立的数据库引擎访问物理数据驱动程序。  
  
 **DDL**  
 数据定义语言。 这些语句在 SQL 定义，而不是处理数据。 例如， **CREATE TABLE**， **CREATE INDEX**，**授予**，以及**撤消**。  
  
 **分隔的标识符**  
 一个包含在标识符引号字符，以便它可以包含特殊字符或关键字 （也称为带引号的标识符） 匹配的标识符。  
  
 **descriptor**  
 一个包含有关列数据或动态参数的信息的数据结构。 未定义的物理表示形式的描述符;应用程序直接访问说明符仅通过其字段操作通过调用 ODBC 函数与描述符句柄。  
  
 **桌面数据库**  
 DBMS 旨在在个人计算机上运行。 通常情况下，这些 Dbms 不提供独立的数据库引擎，必须通过基于文件的驱动程序进行访问。 这些驱动程序中的引擎通常减少对 SQL 和事务的支持。 例如，dBASE、 Paradox、 Btrieve 或 Microsoft® FoxPro®。  
  
 **诊断**  
 包含有关最后一个函数调用的诊断信息的记录使用特定的句柄。 诊断记录与相关联的环境、 连接、 语句和描述符句柄。  
  
 **DML**  
 数据操作语言。 这些操作，而不是定义数据在 SQL 语句。 例如，**插入**，**更新**，**删除**，以及**选择**。  
  
 **driver**  
 例程库公开在 ODBC API 函数。 特定于单个 DBMS 的驱动程序。  
  
 **驱动程序管理器**  
 管理驱动程序对应用程序的访问例程库。 驱动程序管理器加载和卸载 （或连接到和从断开连接） 驱动程序并将传递到正确的驱动程序的 ODBC 函数调用。  
  
 **驱动程序安装程序 DLL**  
 包含特定于驱动程序的安装和配置函数的 DLL。  
  
 **动态游标**  
 可滚动游标可以检测行更新、 删除或插入结果集中。  
  
 **动态 SQL**  
 嵌入式 SQL 中的 SQL 语句创建并在运行时编译的类型。 *另请参阅*静态 SQL。  
  
## <a name="e"></a>E  
 **嵌入式的 SQL**  
 直接在另一种语言，如 COBOL 或 C.ODBC 中编写的程序中包含的 SQL 语句不使用嵌入式的 SQL。 *另请参阅*静态 SQL*和*动态 SQL。  
  
 **环境**  
 全局上下文中访问数据;与环境关联是全局特性，例如，在该环境中的所有连接的列表中的所有信息。  
  
 **环境句柄**  
 包含有关环境的信息的数据结构句柄。  
  
 **escape 子句**  
 中的 SQL 语句的子句。  
  
 **execute**  
 若要运行的 SQL 语句。  
  
## <a name="f"></a>F  
 **fat 游标**  
 *请参阅*块游标。  
  
 **fetch**  
 若要从结果集中检索一个或多个行。  
  
 field   
 *请参阅*列。  
  
 **基于文件的驱动程序**  
 直接访问物理数据驱动程序。 在这种情况下，该驱动程序包含的数据库引擎，并充当驱动程序和数据源。  
  
 **文件数据源**  
 数据源的连接信息存储在.dsn 文件。  
  
 **外键**  
 列或表中匹配另一个表中的主键的列。  
  
 **只进游标**  
 可以仅向前移动在结果集中，通常的游标提取一次只有一个行。 大多数关系数据库支持仅只进游标。  
  
## <a name="h"></a>H  
 **句柄**  
 唯一标识的文件或数据结构类似于一个值。 句柄是仅对创建和使用它们，但传递的其他软件以确定操作的软件有意义。 ODBC 定义环境、 连接、 语句和描述符句的柄。  
  
## <a name="i"></a>I  
 **实现参数描述符 (IPD)**

 **实现行描述符 (IRD)**  
 描述符来描述应用程序指定任何转换前的数据行。  
  
 **安装程序 DLL**  
 安装 ODBC 组件并配置数据源的 DLL。  
  
 **完整性增强功能**  
 SQL 的一个子集用于维护数据库的完整性。  
  
 **接口一致性级别**  
 由驱动程序; 所支持的 ODBC 3.7 接口的级别可以是核心、 级别 1 或 2 级。  
  
 **互操作性**  
 一个应用程序时要使用相同的代码访问不同 Dbms 中的数据的能力。  
  
 **IPD**  
 *请参阅*实现参数描述符 (IPD)。  
  
 **IRD**  
 *请参阅*实现行描述符 (IRD)。  
  
 **ISO/IEC**  
 国际标准组织/国际电工委员会。 ODBC API 基于 ISO/IEC 调用级别接口。

## <a name="k"></a>K

 **keyset**

 **由键集驱动的游标**  
 通过使用由键集来检测更新和删除的行可滚动游标。  
  
## <a name="l"></a>L  
 literal   
 中的 SQL 语句的实际数据值的字符表示形式。  
  
 **锁定**  
 DBMS 将访问限制为多用户环境中的行的过程。 DBMS 通常将位设置对行或包含指示的行的行的物理页或页锁定。  
  
 **长整型数据**  
 通过一定的长度，例如 255 个字节或字符任何二进制或字符数据。 通常更长的时间。 此类数据通常是发送到和从部件中的数据源中检索。 也称为*BLOB*s 或*CLOB*s。  
  
## <a name="m"></a>M  
 **计算机数据源**  
 数据源的连接信息存储在系统 （例如，注册表）。  
  
 **手动提交模式**

 **元数据**  
 描述 SQL 语句中的参数或结果集中的列的数据。 例如，数据类型、 字节长度和精度参数。  
  
 **多层驱动程序**

## <a name="n"></a>N

## <a name="o"></a>O  
 **octet**  
 8 位或 1 个字节。 *另请参阅*字节。  
  
 **八位字节长度**  
 为缓冲区或其包含的数据的八位字节长度。  
  
 **ODBC**  
 开放式数据库连接。 定义一组标准的例程与该应用程序可以访问数据源中的数据的 api 规范。  
  
 **ODBC 管理器**

 打开组  
 发布标准公司。 具体而言，它将发布 SQL 访问组 （压降） 标准。  
  
 **乐观并发**  
 若要提高并发性将不锁定行该策略。 相反，更新或删除它们之前，游标将检查以确定是否已更改它们已由于在上一次读取。 如果是这样，更新或删除将失败。 *另请参阅*保守式并发。  
  
 **外部联接**  
 一种联接中返回的匹配和不匹配的行。 中不匹配的行不匹配的表中的所有列的值设置为 NULL。  
  
 **owner**  
 表的所有者。  
  
## <a name="p"></a>P  
 **parameter**  
 使用参数标记或问号 （？） 标记的 SQL 语句中的变量。 参数被绑定到应用程序变量和执行该语句时，检索其值。  
  
 **参数描述符**  
 介绍用于 SQL 语句，或者由应用程序 （应用程序参数描述符或 APD） 中或之后指定的应用程序 （实现任何转换指定的任何转换之前的运行时参数描述符参数描述符或 IPD）。  
  
 **参数操作数组**  
 包含应用程序可以设置以指示应在忽略相应参数的值的数组**SQLExecDirect**或**SQLExecute**操作。  
  
 **参数状态数组**  
 一个数组，包含参数的状态后调用**SQLExecDirect**或**SQLExecute**。  
  
 **悲观并发**  
 实现可序列化性，以便其他事务不能更改它们锁定行的策略。 *另请参阅*乐观并发*和*可序列化性。  
  
 **定位的操作**  
 作用于当前行的任何操作。 例如，定位 update 和 delete 语句， **SQLGetData**，并**SQLSetPos**。  
  
 **定位的 update 语句**  
 SQL 语句，用于更新当前行中的值。  
  
 **定位的 delete 语句**  
 用于删除当前行的 SQL 语句。  
  
 **prepare**  
 若要编译 SQL 语句。 准备的 SQL 语句创建访问计划。  
  
 **主键**  
 一个或多列唯一标识表中的行。  
  
 procedure   
 一个或多个组预编译的存储为数据库中的命名对象的 SQL 语句。  
  
 **过程列**  
 过程调用，返回的过程中或在创建过程的结果集列的值中的自变量。  
  
## <a name="q"></a>Q

 **查询**  
 SQL 语句。 有时用来表示**选择**语句。  
  
 **quoted identifier**  
 一个包含在标识符引号字符，以便它可以包含特殊字符或关键字 （在 SQL-92 作为分隔标识符中也称为） 匹配的标识符。  
  
## <a name="r"></a>R  
 **radix**  
 数字系统的基数。 通常为 2 或 10。  
  
 **记录**

 **结果集**

 **返回代码**  
 ODBC 函数返回的值。  
  
 **回滚**  
 若要返回到其原始状态事务更改过的值。  
  
 **row**  
 一组描述特定实体的相关列。 也称为*记录*。  
  
 **行描述符**  
 描述符来描述结果集，或者由应用程序 （实现行描述符或 IRD） 或应用程序 （应用程序行描述符或 ARD） 指定任何转换后指定任何转换前的列。  
  
 **行操作数组**  
 包含应用程序可以设置用于指示在中的相应行应忽略的值的数组**SQLSetPos**操作。  
  
 **行状态数组**  
 一个数组，包含行的状态后调用**SQLFetch**， **SQLFetchScroll**，或**SQLSetPos**。  
  
 **rowset**  
 返回读取一次的块状游标行集。  
  
 **行集的缓冲区**  
 缓冲区绑定到列的结果集并在返回整个行集的数据。  
  
## <a name="s"></a>S  
 **SAG**  
 *请参阅*SQL 访问组 （压降）。  
  
 **标量函数**  
 从单一值生成单个值的函数。 例如，更改字符数据的大小写的函数。  
  
 **schema**  
 *请参阅*目录。  
  
 **可滚动游标**  
 可以将向前或向后移动整个结果集游标。  
  
 **可序列化性**  
 是否同时执行两个事务产生串行 （或序列） 执行这些事务相同的结果。 可序列化事务必须维护数据库的完整性。  
  
 **服务器数据库**  
 设计为在客户端/服务器环境中运行 DBMS。 这些 Dbms 提供一个独立的数据库引擎，提供 SQL 和事务的丰富支持。 基于 DBMS 的驱动程序通过访问它们。 例如，Oracle、 Informix、 DB/2 或 Microsoft® SQL Server。  
  
 **set 函数**  
 *请参阅*聚合函数。  
  
 **安装程序 DLL**  
 *请参阅*驱动程序安装程序 DLL*和*转换器安装程序 DLL。  
  
 **单层驱动程序**  
 *请参阅*基于文件的驱动程序。  
  
 **SQL**  
 结构化的查询语言。 一种关系数据库用于查询，语言更新和管理数据。  
  
 **SQL 访问组 （压降）**  
 公司关心 SQL Dbms 一个行业协会。 Open Group 调用级别接口基于最初由 SQL 访问组完成工作。  
  
 **SQL 一致性级别**  
 驱动程序; 支持的 SQL-92 语法级别可以是项、 FIPS 过渡、 中间，或完整。  
  
 **SQL 数据类型**  
 在数据源中存储的列或参数，因为它的数据类型。  
  
 **SQLSTATE**  
 一个五个字符的值，该值指示某个特定错误。  
  
 **SQL 语句**  
 在 SQL 中，以一个关键字开始，并完全描述要执行的操作完成短语。 例如，选择 * 从订单。 不要将 SQL 语句与语句相混淆。  
  
 **State**  
 明确定义符合以下条件的项。 例如，连接了七种状态，包括未分配、 分配、 已连接，且需要的数据。 仅当项处于特定状态时，可以在执行某些操作。 例如，仅当它是处于已分配状态并不是，示例中，当它处于连接状态时，可以释放连接。  
  
 **状态转换**  
 从一个状态到另一个项的移动。 ODBC 定义环境、 连接和语句严苛的状态的转换。  
  
 **语句**  
 与 SQL 语句相关的所有信息的容器。 语句不应与 SQL 语句混淆。  
  
 **语句句柄**  
 包含有关语句的信息的数据结构句柄。  
  
 **静态游标**  
 可滚动游标无法检测到更新、 删除或插入结果集中。 通常由创建结果集的副本。  
  
 **静态 SQL**  
 嵌入式 SQL 中的 SQL 语句是硬编码和编译的程序的其余部分进行编译时类型。 *另请参阅*动态 SQL。  
  
 **存储的过程**  
 *请参阅*过程。  
  
## <a name="t"></a>T  
 **table**  
 行的集合。  
  
 **形式转换**  
 16 位应用程序与 32 位 ODBC 驱动程序一起使用时，转换的 16 位地址到 32 位地址，或反之。  
  
 **transaction**  
 原子工作单元。 必须作为一个整体; 完成在事务中工作如果事务的任何部分失败，则整个事务失败。  
  
 **事务隔离**  
 隔离一个事务影响的所有其他事务的行为。  
  
 **事务隔离级别**  
 事务是隔离的程度的度量值。 有五个事务隔离级别：读取未提交，读取已提交、 可重复读取，可序列化，和版本控制。  
  
 **转换器 DLL**  
 DLL 用于将数据从设置到另一个字符转换。  
  
 **转换器安装程序 DLL**  
 包含特定于转换程序的安装和配置函数的 DLL。  
  
 **两阶段提交**  
 提交两个阶段中的分布式的事务的过程。 在第一个阶段，事务处理器会检查可提交事务的所有部分。 在第二个阶段中，将提交该事务的所有部分。 如果该事务的任何部分指示第一个阶段，它不能为已提交，则不会发生第二个阶段。 ODBC 不支持两阶段提交。  
  
 **类型指示符**  
 一个整数值传递给或从 ODBC 函数以指示应用程序变量、 参数或列的数据类型返回。 ODBC 定义 C 和 SQL 数据类型的类型指示的符。  
  
## <a name="v"></a>V  
 **view**  
 查看一个或多个表中的数据的另一种方法。 视图通常创建为一个或多个表中列的子集。 在 ODBC 中，视图是通常等效于表。
