# <a name="project-settings-migration-mysqltosql"></a>项目设置（迁移）(MySQLToSQL)
迁移页**项目设置**对话框中包含自定义如何 SSMA 将迁移数据从 MySQL 到 SQL Server 的设置。  
  
迁移窗格中现已推出**项目设置**并**默认项目设置**对话框。  
  
-   若要指定所有的 SSMA 项目的设置**工具**菜单中，选择**默认项目设置**，选择项目类型中的**迁移目标版本**的下拉列表想要访问的设置，请单击**常规**的左窗格中，并单击底部**迁移**。  
  
-   若要在指定的当前项目中，设置**工具**菜单中，选择**项目设置**，单击**常规**底部的左窗格中，然后单击**迁移**。  
  
## <a name="options"></a>选项  
  
### <a name="bulk-copy"></a>大容量复制  
  
|术语|定义|  
|--------|--------------|  
|**批大小**|指定批处理大小在数据迁移过程中使用。<br /><br />**默认模式**: 1000年<br /><br />**乐观模式**: 1000年<br /><br />**完整模式**: 1000年|  
|**检查约束**|指定是否将数据插入到 SQL Server 表时，SSMA 应检查约束。<br /><br />**默认模式**: False<br /><br />**乐观模式**: False<br /><br />**完整模式**: False|  
|**激发触发器**|指定当其将数据添加到 SQL Server 表 SSMA 是否应激发插入触发器。<br /><br />**默认模式**: False<br /><br />**乐观模式**: False<br /><br />**完整模式**: False|  
|**保留标识**|指定时它将数据添加到 SQL Server，SSMA 是否保留 MySQL 标识值。 值为 False 将导致由目标分配标识值。<br /><br />**默认模式**: True<br /><br />**乐观模式**: True<br /><br />**完整模式**: True|  
|**保留 Null**|指定是否 SSMA 保留源数据中的 null 值时它将数据添加到 SQL Server 的而不考虑 SQL Server 中指定的默认值。<br /><br />**默认模式**: True<br /><br />**乐观模式**: True<br /><br />**完整模式**: True|  
|**表锁**|指定时它将数据添加到表数据迁移期间，SSMA 是否锁定表。 大容量复制操作期间获取大容量更新锁。 如果值为 False，在行级别设置锁。<br /><br />**默认模式**: False<br /><br />**乐观模式**: False<br /><br />**完整模式**: False|  
  
### <a name="data-modification"></a>数据修改  
  
|术语|定义|  
|--------|--------------|  
|**无效日期迁移**|指定如何迁移使用类似的无效日期 2007年-04-23 或 2000年-06-31 10:00:00 日期和日期时间格式。<br /><br />**默认模式**： 设置 NULL<br /><br />**乐观模式**： 设置 NULL<br /><br />**完整模式**： 设置 NULL|  
|**负时间值迁移**|指定如何迁移类似负值-30:11:00 时间列中。<br /><br />**默认模式**： 设置 NULL<br /><br />**乐观模式**： 设置 NULL<br /><br />**完整模式**： 设置 NULL|  
|**时间值超过了 24 小时迁移**|指定如何迁移时间值的多个"23: 59:59 时间列中。<br /><br />**默认模式**： 设置 NULL<br /><br />**乐观模式**： 设置 NULL<br /><br />**完整模式**： 设置 NULL|  
|**截断要适应列的二进制值**|如果是，SSMA 将截断到 SQL 表的列不适合的二进制值从 MySQL，并生成相应的错误消息。 如果否，则该行将导致错误<br /><br />**默认模式**： 否<br /><br />**乐观模式**： 否<br /><br />**完整模式**： 否|  
|**截断适应列的字符值**|SSMA 会截断到 SQL 表的列不适合的字符值从 MySQL，并生成相应的错误消息。<br /><br />**默认模式**： 否<br /><br />**乐观模式**： 否<br /><br />**完整模式**： 否|  
|**零日期迁移**|指定如何迁移零日期，如 0000-00-00 或 0000-00-00 00:00:00 日期和日期时间列中。<br /><br />**默认模式**： 设置 NULL<br /><br />**乐观模式**： 设置 NULL<br /><br />**完整模式**： 设置 NULL|  
|**零日期迁移**|指定如何迁移具有零个部件等的日期 2009年-01-00 或 2000年-00-00 11:00:00 日期和日期时间列中。<br /><br />**默认模式**： 设置 NULL<br /><br />**乐观模式**： 设置 NULL<br /><br />**完整模式**： 设置 NULL|  
  
### <a name="migration-engine"></a>迁移引擎  
  
|术语|定义|  
|--------|--------------|  
|**迁移引擎**|指定在数据迁移过程中使用的数据库引擎。 客户端数据迁移是指从源和大容量将该数据插入到 SQL Server 中检索数据的 SSMA 客户端。 服务器端数据迁移是指 SSMA 数据迁移引擎 （大容量复制程序） 框中运行 SQL Server 作为 SQL 代理作业从源检索数据并将直接插入 SQL Server，从而避免额外客户端跃点 （更好的性能）。<br /><br />**默认模式**： 客户端数据迁移引擎<br /><br />**乐观模式**： 客户端数据迁移引擎<br /><br />**完整模式**： 客户端数据迁移引擎|  
  
> [!IMPORTANT]  
> 当**迁移引擎**选项设置为**服务器端数据迁移引擎**，将选项设置的新项目**使用 32 位服务器端数据迁移引擎**显示. 它指定是否使用 32 位或 64 位的大容量复制程序 (BCP) 实用程序来迁移数据。  
  
### <a name="misc"></a>杂项  
  
|术语|定义|  
|--------|--------------|  
|**扩展的数据迁移选项**|在单独的详细信息选项卡中显示每个表的额外的数据迁移选项。<br /><br />**默认模式**： 隐藏<br /><br />**乐观模式**： 隐藏<br /><br />**完整模式**： 隐藏|  
|**出错时**|发生错误时停止数据迁移。 它具有三个选项：<br /><br />**停止迁移：** 停止数据迁移操作<br /><br />**转到下一个表：** 停止将数据迁移到当前表和继续到下一个<br /><br />**请继续执行下一批：** 停止将数据迁移到当前批次，并将继续到下一个<br /><br />**默认模式**： 转到下一批<br /><br />**乐观模式**： 转到下一批<br /><br />**完整模式**： 转到下一批|  
  
### <a name="parallel-data-migration"></a>并行数据迁移  
  
|术语|定义|  
|--------|--------------|  
|**并行数据迁移模式下**|指定到分叉线程用来启用并行数据迁移的模式。 在自动模式下，SSMA 选择的线程数 (默认值为 10) 将数据迁移进行分叉。 在自定义模式下，用户可以指定要将数据迁移进行分叉的线程数 （最小值为 1，最大值为 100）。 目前，仅客户端数据迁移引擎支持并行数据迁移。<br /><br />**默认模式**： 自动<br /><br />**乐观模式**： 自动<br /><br />**完整模式**： 自动|  
  
> [!IMPORTANT]  
> 当**并行数据迁移模式下**选项设置为**自定义**，将选项设置的新项目**线程计数**显示。 它指定用于满足数据迁移的线程数。  
  
### <a name="spatial-data"></a>空间数据  
  
|术语|定义|  
|--------|--------------|  
|**处理错误**|指定如何处理迁移的空间数据类型的值中的错误。 如果指定替换为 NULL，则会导致错误的所有空间值将替换为 NULL。 没有替代项是否则完成的。<br /><br />**默认模式**： 生成错误<br /><br />**乐观模式**： 生成错误<br /><br />**完整模式**： 生成错误|  
|**值验证**|指定如何处理无效的空间值。 如果指定了尝试进行有效，正在尝试修改无效的值，使其有效。<br /><br />**默认模式**： 使有效<br /><br />**乐观模式**： 不会更改<br /><br />**完整模式**： 使有效|
