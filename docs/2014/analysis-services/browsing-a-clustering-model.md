# <a name="browsing-a-clustering-model"></a>浏览聚类分析模型
  当你打开聚类分析模型使用**浏览**，在交互式查看器，类似于的聚类分析查看器中显示该模型[!INCLUDE[ssASnoversion](../includes/ssasnoversion-md.md)]。 该查看器可帮助您浏览已创建的分类并了解分类特征。 您还可以将各个段与其他段或填充进行比较和对比。  
  
##  <a name="BKMK_Tabs"></a> 浏览该模型  
 **浏览**窗口包含以下工具来帮助您了解聚类分析模型和浏览基础数据组的属性：  
  
-   [分类关系图](#BKMK_ClusterDiagram)  
  
-   [分类剖面图](#BKMK_ClusterProfiles)  
  
-   [分类特征](#BKMK_ClusterCharacteristics)  
  
-   [分类对比](#BKMK_ClusterDiscrimination)  
  
 若要体验与聚类分析模型，可以使用示例数据的工作簿，培训选项卡上的示例数据并生成聚类分析模型使用[群集向导&#40;for Excel 的数据挖掘外接&#41;](cluster-wizard-data-mining-add-ins-for-excel.md)和所有默认值.  
  
###  <a name="BKMK_ClusterDiagram"></a> 分类关系图  
 **分类关系图**选项卡显示挖掘模型中的所有分类。 在这里，可以查看数据集中有多少个不同分组，以及这些分组之间的远近。  
  
##### <a name="explore-the-cluster-diagram"></a>探索分类关系图  
  
1.  单击关系图中的“分类 1”。  
  
     注意连接所有分类的灰色线条如何变化以便以浅蓝色突出显示指向所选分类的线条。  
  
     ![群集关系图简介](media/dm13-cluster-diagram-intro.gif "群集关系图简介")  
  
     连接到另一个群集的行的强度表示群集的相似性的强度。 如果明暗度较浅或无明暗度，则表示分类的相似程度较低。 连线颜色越深，表示两个分类之间的相似性越大。  
  
2.  单击并拖动分类关系图左侧的滑块可调整查看器显示的线条数目。  
  
     向下拖动滑块时，仅显示分类间的最强链接。 这有助于您重点关注相关的组。  
  
3.  请注意**明暗度变量**的右上角的控件**分类关系图**窗口。  
  
     默认情况下，它设置为**填充**。 这表示颜色较暗的分类具有更高程度的支持。  
  
4.  将鼠标指针停留在任一分类上。  
  
     将显示工具提示，其中包含该分类的总体。  
  
5.  现在，单击**明暗度变量**下拉列表并选择**年龄**变量。 执行此操作时，将在显示的值列表**状态**文本框。  
  
     用作此模型的输入的“Age”列包含连续数值，但为了聚类分析，该算法始终将数字离散化。 你可以看到的箱或组，该算法创建，如"非常低 (\<= 27)"和"非常高 (> = 63)"。  
  
6.  从**状态**下拉列表中，选择**非常高**并查看关系图将如何变化。  
  
     通过更改明暗度变量，可以一目了然地看到哪些分类包含此目标年龄组内的更多客户，哪些分类包含此年龄组内的很少客户。  
  
     ![修改群集关系图，以显示年龄](media/dm13-cluster-diagramshadebyage.gif "修改分类关系图，以显示期限")  
  
     明暗度越深，所聚类的目标属性和值分配比例就越高。  
  
7.  找到阴影的群集最暗在**明暗度变量**设置为 Age > 65。  
  
     将鼠标指针悬停在该分类上。  
  
     工具提示中的值此时显示年龄超过 65 的此分类中的客户人数。  
  
8.  右键单击该群集，然后选择**重命名群集**。 键入新名称，具有说明性，如**通过 65**。 新名称随模型保存到服务器，可用于标识其他聚类分析视图中的分类。  
  
 [返回页首](#BKMK_Tabs)  
  
###  <a name="BKMK_ClusterProfiles"></a> 分类剖面图  
 **分类剖面图**选项卡，可以比较组成部分的所有群集一目了然。 要熟悉模型，这里是很好的起点。 如果您在探索特定分类并且需要查找相关分类，稍后此视图也非常有用。  
  
 **分类剖面图**还为你提供的群集的各个不同的方式的很好概述。 因此，您可能会发现使用此视图为每个分类提供说明性名称十分方便。  
  
##### <a name="explore-the-cluster-profiles"></a>探索分类剖面图  
  
1.  中的职业，单击单元格**状态**列，以查看有关职业的所有值的列表。  
  
2.  现在，将鼠标指针移到分类剖面图中的“Occupation”上。  
  
     工具提示显示该分类中的职业分布。  
  
     ![在工具提示中或在图例中查看详细的值](media/dm13-cluster-profile-age-tooltip.gif "在工具提示中或在图例中查看详细的值")  
  
     请注意，在某些群集 （例如在图中的那一个） 的职业列表并不完整，并且某些职业替换为标签，**其他**。  
  
     这是设计使然，因为可能很难判断直方图中许多小条之间的差别。 默认情况下仅重要性最高的条得到保留，并且其余图条则组合到灰色**其他**存储桶。  
  
     若要更改的任何直方图中可见的条数，请使用选项**直方图条数**。  
  
3.  请注意，**年龄**列看起来不同于其他。 单击图表中用于表示 Age 的菱形。  
  
     Age 列最初仅包含连续值。 聚类分析算法需要离散值，因此，它基于值的分布将 Age 列中的数值分组为有限数目的年龄组。  
  
4.  单击分类剖面图中的一个菱形图表。  
  
     只有在源数据使用连续数值时，才会显示这些菱形图表。 菱形图表提供了一些有用的说明性统计信息，包括每个分类中该值的平均偏差和标准偏差：  
  
    -   菱形图表中的线条表示该属性的值范围。 值还会显示在**状态**左侧列**配置文件**图表。  
  
    -   菱形的中心位于该节点的平均值处。  
  
    -   菱形的宽度表示该节点处属性的方差。 因此，较细的菱形表示该节点可以生成更精确的预测。  
  
5.  要在关系图中进行更多的空间，请右键单击群集，你不需要立即，查看并选择**隐藏列**。 此操作不会将该列从模型中删除，而只是将其临时折叠。  
  
     若要查看已隐藏的群集，可以单击和拖动列边缘，或者从列表中，选择群集名称**更多群集**。  
  
6.  向下滚动属性列表，直至找到“Bike Buyer”，然后查找具有最高“Yes”值百分比的分类。  
  
     右键单击你想要重命名，选择该群集的列标题**重命名群集**，和类型**自行车购买者**。  
  
     新的分类名称保留在所有视图中以及服务器上，直到您重新处理该模型。  
  
     ![重命名要使图表易于使用的群集](media/dm13-cluster-rename.gif "重命名要使图表易于使用的群集")  
  
 **提示**  
  
-   单击列标题，可以将列中的属性按照其对分类的重要性来进行排序。  
  
-   在查看器中拖动各列以对它们重新排序。  
  
-   单击要查看详细统计信息中的配置文件图表中的任意单元**挖掘图例**。  
  
-   右键单击任何单元格，然后选择**钻取模型列**若要将基础数据输出到 Excel 中的新工作表。  
  
-   右键单击群集的列标题并选择**钻取到结构数据**以获取有关群集成员未包含在模型的详细的信息。  
  
     例如，如果要分析客户，则可能要在基础数据（挖掘结构）中保留联系信息而不将数据包括在模型中，因为这种数据对分析没有用处。 但是，在将客户分配给分类后，您可通过使用钻取来查看详细数据。  
  
 [返回页首](#BKMK_Tabs)  
  
###  <a name="BKMK_ClusterCharacteristics"></a> 分类特征  
 通过“分类特征”视图，您实际上可以探索单个分类，以了解哪些属性最能概括此组数据的特征。  
  
##### <a name="explore-the-cluster-characteristics"></a>探索分类特征  
  
1.  选择**通过 65**群集从**群集**列表。  
  
     选择分类后，您可以详细查看构成特定分类的特征。  
  
     分类包含的属性列在 **“变量”** 列中，所列属性的状态列在 **“值”** 列中。  
  
     重要性，伴随及其概率表示彩色条中为此群集中的顺序列出属性状态**概率**列。  
  
     ![聚类分析模型的特征](media/dm13-cluster-characteristics.gif "聚类分析模型的特征")  
  
2.  单击**变量**作为排序依据属性列。  
  
     通过更改排序变量，您可以更方便地查看变量值（如收入或汽车拥有情况）是如何在该组中分布的。  
  
3.  单击**复制到 Excel**。  
  
     一个新工作表将添加到包含所选分类的特征的工作簿中。  
  
4.  现在从列表中，选择不同的群集**自行车购买者**。  
  
5.  单击**复制到 Excel**。  
  
     请注意，新的分类特征图表将会添加到其自己的工作表中。 您可以将该图表移到与其他配置文件相同的工作表上，以便在下一个步骤中更方便地对它们进行比较。  
  
 **提示**  
  
-   请注意，“Over 65”分类中客户的主要特征是他们不购买您的产品！ 如果需要了解原因，可以浏览各分类并比较各组，也可以使用适合探索原因与结果的算法来创建一个相关模型，如决策树模型或 Naïve Bayes 模型。  
  
-   如果需要获得此分类（或所有分类）的属性与概率的完整列表，可以创建一个查询。 聚类分析模型的查询的示例，请参阅[聚类分析模型查询示例](data-mining/clustering-model-query-examples.md)。  
  
 [返回页首](#BKMK_Tabs)  
  
###  <a name="BKMK_ClusterDiscrimination"></a> 分类对比  
 你使用**分类对比**选项卡来比较两个群集之间或群集和在数据集中的所有其他情况之间的属性。  
  
 以突出显示此查看器的功能，我们将比较其根据你创建的 Excel 中的并排显示表**分类特征**视图。  
  
##### <a name="explore-cluster-discrimination"></a>探索分类对比  
  
1.  使用 **“分类 1”** 和 **“分类 2”** 列表选择要比较的分类。  
  
    -   对于“分类 1”请选择“Over 65”。  
  
    -   对于“分类 2”，请选择“Bike Buyers”。  
  
     比较结果如下图所示。  
  
     ![比较在模型中的群集](media/dm13-cluster-compareclusters.gif "比较模型中的群集")  
  
     请注意，事实上，**分类对比**查看器将复杂的查询发送到数据挖掘服务器，以提取中区分两个组，使其更容易比较各个最重要的属性两个设置的客户。  
  
2.  单击任一**优先...** 列。  
  
     属性和值表右侧的条显示了哪些功能或值最适合作为所选分类的特征。  
  
3.  现在，在 Excel 中对各列表进行比较。  
  
     ![关联模型的依赖关系网络图](media/dm13-comparing-profiles-in-excel.gif "关联模型的依赖关系网络图")  
  
     因为用于在查看器中生成图像的基础统计信息以表的形式保存到 Excel，所以，您可以筛选、排序和查看实际概率值。  
  
     除使用 Excel 外，建议您尝试将分类查看器用于 Visio，这样不仅可以查看数据点，还可以广泛修改和增强图形。 有关详细信息，请参阅[群集关系图演练&#40;数据挖掘外接程序&#41;](cluster-diagram-walkthrough-data-mining-add-ins.md)。  
  
 **提示**  
  
 获取后分组情况下，客户的一些见解，请尝试使用[模拟方案&#40;for Excel 表分析工具&#41;](what-if-scenario-table-analysis-tools-for-excel.md)或[目标查找方案&#40;for Excel 表分析工具&#41;](goal-seek-scenario-table-analysis-tools-for-excel.md)工具，若要浏览的模型中可能更改会影响结果的因素。
