# <a name="installing-ssma--for-sybase-sybasetosql"></a>安装适用于 Sybase (SybaseToSQL) SSMA
[!INCLUDE[msCoName](../../includes/msconame_md.md)][!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]Migration Assistant (SSMA) 用于 Sybase 包含的客户端应用程序使用来执行从 Sybase 自适应 Server Enterprise (ASE) 中，到的迁移[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]或 SQL Azure。 它还包含已迁移数据库中支持数据迁移和 ASE 系统函数的使用的扩展包。  
  
你将在其中执行迁移步骤的计算机上安装客户端应用程序。 你必须安装在正在运行的计算机上的扩展包文件[!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]将托管已迁移的数据库的位置。  
  
## <a name="upgrading-ssma-for-sybase"></a>升级用于 Sybase 的 SSMA  
如果你想要升级到更高版本的 SSMA for Sybase，您必须首先卸载客户端和服务器扩展包，，然后安装较新版本。  
  
如果 Sybase 的 SSMA 早期版本中打开项目，SSMA 会询问你是否想要将项目转换为较新版本。 你必须单击**是**用于 SSMA 的较新版本中的项目。
