# <a name="odbc-glossary"></a>ODBC 术语表
## <a name="a"></a>A  
 **访问计划**  
 数据库引擎为执行 SQL 语句而生成的计划。 等效于从第三代语言（如 C）编译的可执行代码。  
  
 **聚合函数**  
 从一组值生成单个值的函数，通常与**GROUP BY**和**HAVING**子句一起使用。 聚合函数包括**SUM**AVG、COUNT、MAX、MIN 和 SUM 。 **AVG** **COUNT** **MAX** **MIN** 也称为*集函数*。 *另请参阅*标量函数。  
  
 **Ansi**  
 美国国家标准协会。 ODBC API 基于 ANSI 呼叫级接口。  
  
 **APD**  
 *请参阅*应用程序参数描述符 （APD）。  
  
 **API**  
 应用程序编程接口。 应用程序用于请求和执行较低级别的服务的一组例程。 ODBC API 由 ODBC 函数组成。  
  
 **应用程序**  
 调用 ODBC API 中的函数的可执行程序。  
  
 **应用程序参数描述符 （APD）**  
 描述在应用程序指定的任何转换之前 SQL 语句中使用的动态参数的描述符。  
  
 **应用程序行描述符 （ARD）**  
 表示应用程序缓冲区中的列元数据和数据的描述符，描述应用程序指定的任何数据转换之后的一行数据。  
  
 **阿尔德**  
 *请参阅*应用程序行描述符 （ARD）。  
  
 **自动提交模式**  
 事务提交模式，其中事务在执行后立即提交。  
  
## <a name="b"></a>B  
 **行为变化**  
 某些功能从 ODBC *3.x*行为更改为 ODBC *2.x*行为，反之亦然。 由更改SQL_ATTR_ODBC_VERSION环境属性引起的。  
  
 **二进制大对象 （BLOB）**  
 超过一定字节数（如 255）的任何二进制数据。 通常更长。 此类数据通常以部分的方式发送到数据源并从数据源中检索。 也称为*长数据*。  
  
 **绑定**  
 作为动词，将结果集中的列或 SQL 语句中的参数与应用程序变量相关联的行为。 作为名词，协会。  
  
 **绑定偏移量**  
 添加到所有绑定列或参数数据的数据缓冲区地址和长度/指示器缓冲区地址的值，生成新地址。  
  
 **块状游标 (block cursor)**  
 一次能够获取多行数据的游标。  
  
 **缓冲区**  
 用于在应用程序和驱动程序之间传递数据的应用程序内存。 缓冲区通常成对：*数据缓冲区*和*数据长度缓冲区*。  
  
 byte   
 八位或一个八位。 *另请参阅*八进制。  
  
## <a name="c"></a>C  
 **C 数据类型**  
 C 程序中变量的数据类型，在这种情况下，是应用程序。  
  
 **目录**  
 描述数据库形状的数据库中的系统表集。 也称为*架构*或*数据字典*。  
  
 **目录功能**  
 用于从数据库的目录中检索信息的 ODBC 函数。  
  
 **CLI**  
 *参见*Api。  
  
 **客户端/服务器**  
 一种数据库访问策略，其中一个或多个客户端通过服务器访问数据。 客户端通常实现用户界面，而服务器控制数据库访问。  
  
 **列**  
 行中单个信息项的容器。 也称为*字段*。  
  
 **提交**  
 使事务中的更改永久化。  
  
 **并发**  
 多个事务同时访问相同数据的能力。  
  
 **一致性级别**  
 驱动程序或数据源支持的离散功能集。 ODBC 定义 API 一致性级别和 SQL 一致性级别。  
  
 **连接**  
 驱动程序和数据源的特定实例。  
  
 **连接浏览**  
 在网络中搜索要连接到的数据源。 连接浏览可能涉及几个步骤。 例如，用户可能首先浏览网络以获取服务器，然后浏览特定服务器以获取数据库。  
  
 **连接句柄**  
 包含有关连接信息的数据结构的句柄。  
  
 **当前行**  
 光标当前指向的行。 定位操作在当前行上作用。  
  
 **光标**  
 将数据行返回给应用程序的软件。 可能是以计算机终端上的闪烁光标命名的;正如该光标指示屏幕上的当前位置一样，结果集中的光标指示结果集中的当前位置。  
  
## <a name="d"></a>D  
 **数据缓冲区**  
 用于传递数据的缓冲区。 通常与数据缓冲区关联的是*数据长度缓冲区*。  
  
 **数据字典 (data dictionary)**  
 *请参阅*目录。  
  
 **数据长度缓冲区**  
 用于传递相应*数据缓冲区*中值长度的缓冲区。 数据长度缓冲区还用于存储指示器，例如数据值是否为 null 终止。  
  
 **数据源**  
 用户要访问的数据及其关联的操作系统、DBMS 和网络平台（如果有）。  
  
 **数据类型**  
 数据片段的类型。 ODBC 定义 C 和 SQL 数据类型。 *另请参阅*类型指示器。  
  
 **执行时的数据列**  
 调用**SQLSetPos**后为其发送数据的列。 因此命名是因为数据在执行时发送，而不是放置在行集缓冲区中。 长数据通常在执行时以零件发送。  
  
 **执行时的数据参数**  
 调用**SQLExecute**或**SQLExecDirect**后发送数据的参数。 之所以如此命名，是因为数据是在执行 SQL 语句而不是放置在参数缓冲区中时发送的。 长数据通常在执行时以零件发送。  
  
 **数据库**  
 DBMS 中的离散数据集合。 也是 DBMS。  
  
 **数据库引擎**  
 DBMS 中用于解析和执行 SQL 语句并访问物理数据的软件。  
  
 **Dbms**  
 数据库管理系统。 物理数据库和用户之间的软件层。 该 DBMS 管理对数据库的所有访问。  
  
 **基于 DBMS 的驱动程序**  
 通过独立数据库引擎访问物理数据的驱动程序。  
  
 **DDL**  
 数据定义语言。 SQL 中定义而不是操作数据的语句。 例如，**创建表**、**创建索引**、**授予**和**撤销**。  
  
 **分隔标识符 (delimited identifier)**  
 包含在标识符报价字符中的标识符，以便它可以包含特殊字符或匹配关键字（也称为报价标识符）。  
  
 **描述 符**  
 保存有关列数据或动态参数的信息的数据结构。 描述符的物理表示形式未定义;因此，描述符的物理表示形式未定义。应用程序只能通过使用描述符句柄调用 ODBC 函数来操作其字段，才能直接访问描述符。  
  
 **桌面数据库**  
 设计为在个人计算机上运行的 DBMS。 通常，这些 DBMS 不提供独立的数据库引擎，必须通过基于文件的驱动程序进行访问。 这些驱动程序中的引擎通常减少了对 SQL 和事务的支持。 例如，dBASE、悖论、Btrieve 或微软®福克斯专业®。  
  
 **诊断**  
 包含有关使用特定句柄的最后一个函数的诊断信息的记录。 诊断记录与环境、连接、语句和描述符句柄相关联。  
  
 **DML**  
 数据操作语言。 SQL 中的那些语句操作数据而不是定义数据。 例如，**插入**、**更新**、**删除**和**选择**。  
  
 **司机**  
 一个例程库，用于公开 ODBC API 中的函数。 驱动程序特定于单个 DBMS。  
  
 **驱动程序管理器**  
 管理对应用程序驱动程序的访问的例程库。 驱动程序管理器加载和卸载（或连接到和断开）驱动程序，并将对 ODBC 函数的调用传递给正确的驱动程序。  
  
 **驱动程序设置 DLL**  
 包含特定于驱动程序的安装和配置功能的 DLL。  
  
 **动态游标 (dynamic cursor)**  
 能够检测在结果集中更新、删除或插入的行的可滚动光标。  
  
 **动态 SQL**  
 在其中创建和编译 SQL 语句的嵌入式 SQL 类型。 *另请参阅*静态 SQL。  
  
## <a name="e"></a>E  
 **嵌入式 SQL**  
 直接包含在用其他语言（如 COBOL 或 C. ODBC）编写的程序中的 SQL 语句不使用嵌入式 SQL。 *另请参阅*静态 SQL*和*动态 SQL。  
  
 **环境**  
 访问数据的全局上下文;与环境关联的是任何全局信息，例如该环境中所有连接的列表。  
  
 **环境句柄**  
 包含有关环境信息的数据结构的句柄。  
  
 **转义条款**  
 SQL 语句中子句。  
  
 **执行**  
 运行 SQL 语句。  
  
## <a name="f"></a>F  
 **脂肪光标**  
 *请参阅*块光标。  
  
 **获取**  
 从结果集中检索一个或多个行。  
  
 **field**  
 *请参阅*列。  
  
 **基于文件的驱动程序**  
 直接访问物理数据的驱动程序。 在这种情况下，驱动程序包含数据库引擎，并同时充当驱动程序和数据源。  
  
 **文件数据源**  
 存储连接信息在 .dsn 文件中的数据源。  
  
 **外键**  
 表中匹配另一个表中的主键的列。  
  
 **仅转发光标**  
 只能向前移动的结果集，并且通常一次只能获取一行的游标。 大多数关系数据库仅支持仅转发游标。  
  
## <a name="h"></a>H  
 **处理**  
 唯一标识文件或数据结构等内容的值。 句柄仅对创建和使用它们的软件有意义，但由其他软件传递以识别内容。 ODBC 定义环境、连接、语句和描述符的句柄。  
  
## <a name="i"></a>I  
 **实现参数描述符 （IPD）**  
 描述在应用程序指定的任何转换后 SQL 语句中使用的动态参数的描述符。  
  
 **实现行描述符 （IRD）**  
 描述应用程序指定的任何转换之前的一行数据的描述符。  
  
 **安装程序 DLL**  
 安装 ODBC 组件并配置数据源的 DLL。  
  
 **诚信增强设施**  
 旨在维护数据库完整性的 SQL 子集。  
  
 **接口一致性级别**  
 驱动程序支持的 ODBC 3.7 接口的级别;可以是核心、级别 1 或级别 2。  
  
 **互 操作 性**  
 一个应用程序在访问不同 DBMS 中的数据时使用相同的代码的能力。  
  
 **IPD**  
 *参见*实现参数描述符 （IPD）。  
  
 **税务局**  
 *参见*实现行描述符 （IRD）。  
  
 **ISO/IEC**  
 国际标准组织/国际电工委员会。 ODBC API 基于 ISO/IEC 呼叫级接口。  
  
## <a name="j"></a>J  
 **加入**  
 关系数据库中的一个操作，通过匹配指定列中的值来链接两个或多个表中的行。  
  
## <a name="k"></a>K  
 **关键**  
 其值标识行的列。 *另请参阅*外键*和*主键。  
  
 **键集**  
 混合或按键集驱动的游标用于重新提取行的一组键。  
  
 **由键集驱动的游标 (keyset-driven cursor)**  
 使用键集检测更新和删除的行的可滚动游标。  
  
## <a name="l"></a>L  
 **文字**  
 SQL 语句中实际数据值的字符表示形式。  
  
 **锁定**  
 DBMS 限制对多用户环境中的行的访问的过程。 DBMS 通常设置一行或物理页上的位，其中包含指示该行或页面已锁定的行。  
  
 **长数据**  
 超过特定长度的任何二进制或字符数据，例如 255 字节或字符。 通常更长。 此类数据通常以部分的方式发送到数据源并从数据源中检索。 也称为*BLOB*或*CLOB。*  
  
## <a name="m"></a>M  
 **机器数据源**  
 存储连接信息的数据源（例如注册表）。  
  
 **手动提交模式**  
 事务提交模式，其中事务必须通过调用**SQLTransact**显式提交。  
  
 **元**  
 描述 SQL 语句中的参数或结果集中的列的数据。 例如，参数的数据类型、字节长度和精度。  
  
 **多层驱动程序**  
 *参见*基于 DBMS 的驱动程序。  
  
## <a name="n"></a>N  
 **空值**  
 没有显式分配的值。 特别是，NULL 值不同于零或空白。  
  
## <a name="o"></a>O  
 **八点**  
 八位或一个字节。 *另请参阅*字节。  
  
 **八进制长度**  
 缓冲区的长度（以八位字节为单位）或其包含的数据。  
  
 **ODBC**  
 打开数据库连接。 API 的规范，用于定义应用程序可以访问数据源中数据的标准例程集。  
  
 **ODBC 管理器**  
 调用安装程序 DLL 以配置数据源的可执行程序。  
  
 打开组  
 发布标准的公司。 特别是，它发布 SQL 访问组 （SAG） 标准。  
  
 **乐观并发**  
 增加行未锁定的并发的策略。 相反，在更新或删除它们之前，光标会检查它们自上次读取以来是否已更改。 如果是这样，更新或删除将失败。 *另请参阅*悲观并发。  
  
 **外部联接**  
 返回匹配行和非匹配行的联接。 不匹配行中不匹配表中所有列的值都设置为 NULL。  
  
 **所有者**  
 表的所有者。  
  
## <a name="p"></a>P  
 **参数**  
 SQL 语句中的变量，标有参数标记或问号 （？）。 参数绑定到应用程序变量，并在执行语句时检索其值。  
  
 **参数描述符**  
 描述 SQL 语句中使用的运行时参数的描述符，无论是在应用程序指定的任何转换（应用程序参数描述符或 APD）之前，还是在应用程序指定的任何转换（实现参数描述符或 IPD）之后。  
  
 **参数操作数组**  
 包含应用程序可以设置为指示应在**SQLExecDirect**或**SQLExecute**操作中忽略相应参数的值的数组。  
  
 **参数状态数组**  
 在调用**SQLExecDirect**或**SQLExecute**后包含参数状态的数组。  
  
 **保守式并发 (pessimistic concurrency)**  
 实现序列化的策略，其中行被锁定，以便其他事务无法更改它们。 *另请参阅*乐观并发*性和*序列化性。  
  
 **定位操作**  
 对当前行运行的任何操作。 例如，定位更新和删除语句 **、SQLGetData**和**SQLSetPos**。  
  
 **定位更新语句**  
 用于更新当前行中的值的 SQL 语句。  
  
 **定位删除语句**  
 用于删除当前行的 SQL 语句。  
  
 **准备**  
 编译 SQL 语句。 通过准备 SQL 语句创建访问计划。  
  
 **主键**  
 唯一标识表中行的列。  
  
 **程序**  
 一组一个或多个预编译的 SQL 语句，这些语句作为命名对象存储在数据库中。  
  
 **过程列**  
 过程调用中的参数、过程返回的值或过程创建的结果集中的列。  
  
## <a name="q"></a>Q  
 **限定符 (Qualifier)**  
 包含一个或多个表的数据库。  
  
 **查询**  
 SQL 语句。 有时用于表示**SELECT**语句。  
  
 **quoted identifier**  
 包含在标识符报价字符中的标识符，以便它可以包含特殊字符或匹配关键字（在 SQL-92 中也称为分隔标识符）。  
  
## <a name="r"></a>R  
 **拉迪克斯**  
 数字系统的基础。 通常为 2 或 10。  
  
 **记录**  
 *请参阅*行。  
  
 **结果集**  
 通过执行**SELECT**语句创建的行集。  
  
 **返回代码**  
 ODBC 函数返回的值。  
  
 **回滚 (roll back)**  
 将事务更改的值返回到其原始状态。  
  
 **行**  
 描述特定实体的一组相关列。 也称为*记录*。  
  
 **行描述符**  
 描述结果集列的描述符，在应用程序指定的任何转换（实现行描述符或 IRD）之前，或在应用程序指定的任何转换（应用程序行描述符或 ARD）之后。  
  
 **行操作数组**  
 包含应用程序可以设置为指示应在**SQLSetPos**操作中忽略相应行的值的数组。  
  
 **行状态数组**  
 在调用**SQLFetch、SQLFetchScroll**或**SQLSetPos**后包含行状态的数组。 **SQLFetchScroll**  
  
 **排集**  
 块游标在单个提取中返回的行集。  
  
 **排集缓冲区**  
 绑定到结果集的列的缓冲区，并在其中返回整个行集的数据。  
  
## <a name="s"></a>S  
 **凹陷**  
 *参见*SQL 访问组 （SAG）。  
  
 **标量函数**  
 从单个值生成单个值的函数。 例如，更改字符数据的项例的函数。  
  
 **架构**  
 *请参阅*目录。  
  
 **可滚动光标**  
 可在结果集中向前或向后移动的光标。  
  
 **序列化**  
 是否同时执行两个事务会产生与这些事务的串行（或顺序）执行相同的结果。 需要序列化事务来维护数据库完整性。  
  
 **服务器数据库**  
 设计为在客户端/服务器环境中运行的 DBMS。 这些 DBMS 提供了一个独立的数据库引擎，为 SQL 和事务提供丰富的支持。 它们通过基于 DBMS 的驱动程序进行访问。 例如，Oracle、Informix、DB/2 或 Microsoft ® SQL 服务器。  
  
 **设置功能**  
 *请参阅*聚合函数。  
  
 **设置 DLL**  
 *请参阅*驱动程序设置 DLL*和*转换器设置 DLL。  
  
 **单层驱动程序**  
 *请参阅*基于文件的驱动程序。  
  
 **SQL**  
 结构化查询语言。 关系数据库用于查询、更新和管理数据的语言。  
  
 **SQL 访问组 （SAG）**  
 与 SQL DBMS 相关的公司行业联盟。 开放组的呼叫级接口基于最初由 SQL 访问组完成的工作。  
  
 **SQL 符合性级别**  
 驱动程序支持的 SQL-92 语法级别;可以是入口、FIPS 过渡、中间或完整。  
  
 **SQL 数据类型**  
 列或参数的数据类型，因为它存储在数据源中。  
  
 **SQLSTATE**  
 指示特定错误的五个字符值。  
  
 **SQL 语句**  
 SQL 中以关键字开头并完全描述要执行的操作的完整短语。 例如，选择 = 从订单。 SQL 语句不应与语句混淆。  
  
 **状态**  
 项的已定义条件。 例如，连接有七种状态，包括未分配、分配、连接和需要的数据。 仅当项处于特定状态时，才能执行某些操作。 例如，仅当连接处于分配状态时，才能释放连接，例如，当连接处于连接状态时，才能释放。  
  
 **状态转换**  
 项从一种状态到另一种状态的移动。 ODBC 为环境、连接和语句定义严格的状态转换。  
  
 **声明**  
 与 SQL 语句相关的所有信息的容器。 语句不应与 SQL 语句混淆。  
  
 **语句句柄**  
 包含语句信息的数据结构的句柄。  
  
 **静态游标 (static cursor)**  
 无法检测结果集中的更新、删除或插入的可滚动光标。 通常通过复制结果集实现。  
  
 **静态 SQL**  
 一种嵌入式 SQL，其中 SQL 语句在编译程序的其余部分时进行硬编码和编译。 *另请参阅*动态 SQL。

## <a name="t"></a>T  
 **表**  
 行的集合。  
  
 **特恩金**  
 当 16 位应用程序与 32 位 ODBC 驱动程序一起使用时，将 16 位地址转换为 32 位地址，反之亦然。  
  
 **交易**  
 一个工作的原子单元。 必须将事务中的工作作为一个整体来完成；如果该事务的任何部分失败，那么整个事务都会失败。  
  
 **事务隔离**  
 将一个事务与所有其他事务的影响隔离开来的行为。  
  
 **事务隔离级别**  
 衡量事务隔离程度的度量。 有五个事务隔离级别：读取未提交、已读、可重复读取、可序列化和版本控制。  
  
 **译者 DLL**  
 用于将数据从一个字符集转换为另一个字符集的 DLL。  
  
 **转换器设置 DLL**  
 包含特定于转换器的安装和配置功能的 DLL。  
  
 **两阶段提交**  
 分两个阶段提交分布式事务的过程。 在第一阶段，事务处理器检查事务的所有部分是否可以提交。 在第二阶段，事务的所有部分都提交。 如果事务的任何部分在第一阶段指示无法提交，则第二阶段不会发生。 ODBC 不支持两阶段提交。  
  
 **类型指示器**  
 传递给或从 ODBC 函数返回的整数值，用于指示应用程序变量、参数或列的数据类型。 ODBC 为 C 和 SQL 数据类型定义类型指示器。  
  
## <a name="v"></a>V  
 **视图**  
 查看一个或多个表中的数据的另一种方法。 视图通常创建为一个或多个表中的列的子集。 在 ODBC 中，视图通常等效于表。
