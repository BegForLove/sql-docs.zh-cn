# <a name="hardware-installation"></a>硬件安装
本主题介绍如何将移动，请解压缩，并安装 SQL Server PDW 设备的硬件。 本主题仅供参考，旨在帮助你了解这一过程。 应打开包装后，安装，以及移交给你的它之前，请验证你的设备。 客户参与是必需的项目例如数据中心访问、 电源和以太网连接。  
  
## <a name="BeforeMoving"></a>从加载停靠移动任何组件之前  
移动、 解包，或机架的任何设备组件之前，请执行以下任务。  
  
|任务|Description|  
|--------|---------------|  
|验证所有组件均已都到达|材料清单 (BOM) 用于验证所有组件均已到达，且您的数据中心接收的停靠在其托盘上。|  
|验证数据中心满足所有要求的设备|通过查看的硬件规格和电缆你 IHV 所提供的关系图来启动此任务。 接下来的步骤提供有关机架的具体信息空间和连接要求。|  
|验证数据中心具有正确的机架空间|验证数据中心具有足够的空间用于所有设备机架。<br /><br />验证为空且准备好接收设备机架机架空间。|  
|验证数据中心满足连接要求|验证数据中心满足缆线关系图中的缆线连接要求。<br /><br />验证，将为所有电缆和电源线连接留出空间后架装设备节点。|  
|验证停靠和机架之间楼层符合权重要求|验证所有地板托盘和机架之间可以都支持的权重的设备节点中，尤其是在具有引发下限的数据中心。<br /><br />有关权重上的每个组件的信息，请与你 IHV 联系。|  
|安全数据中心的机架|安全数据中心中的位置对于你数据中心位置，如地震传送带容易导致地震的地理区域中，根据需要使用其他设备的机架。|  
|准备进行传输组件的帮助|事先确定哪些协助、 设备和将需要处理每个组件，安全地和而不会导致损坏的工具。|  
  
## <a name="Moving"></a>将从加载停靠的机架移到数据中心  
每个托盘包含一个设备机架，包括节点、 电缆、 线连接等的所有组件。  
  
使用以下清单帮助将每个设备机架从加载停靠在托盘移到其在数据中心中的机架位置。 首先，将移动控件机架，然后移动其他设备数据机架。  
  
> [!WARNING]  
> 严格按所述执行这些步骤失败可能导致身体伤害，对你的 SQL Server PDW 设备或其他问题造成损坏。  
>   
> 从不尝试提升或移动设备节点或其他大量组件而无需获取帮助或正确的设备。 联系你 IHV 了解权重上的每个组件的信息，以便你可以提前确定哪些协助、 设备和将需要处理每个组件，安全地和而不会导致损坏的工具。  
  
|任务|Description|  
|--------|---------------|  
|验证托盘级别|在开始移动或解压缩托盘之前，请确保位于级别接地。|  
|Unbolt 托盘从节点|从顶部的托盘开始，unbolt 托盘中的顶级节点。|  
|将节点移到运输车或购物车可支持权重|使用渐变和提起/移动技术的正确状态，可以将节点移到运输车或可以支持权重的车。|  
|到数据中心传输节点|使用正确提起/移动方法来将该节点移动到在数据中心机架中的位置。|  
|安全机架的数据中心中的节点|机架安全中的数据中心的位置中的节点。|  
|下一步的节点或组件重复这些步骤|重复上述步骤以将下一个节点或其他设备组件移到数据中心。|  
  
## <a name="AfterMoving"></a>安装其他组件  
使用以下清单来安装其他组件。  
  
|任务|Description||  
|--------|---------------|-|  
|解压缩和机架网络交换机和 Pdu|使用机架图在机架中的适当位置放置的网络交换机和 Pdu。||  
|连接根据电缆标签 Infiniband 和以太网电缆|请参阅缆线连接的图。 每条电缆指定它需要连接每一端上具有的标签。||  
|将所有电源线连接|请参阅缆线连接的图。||  
|打开到机架和 Pdu 电源|连接电源的机架和从 Pdu 到机架。 **不要电源上的任何其他设备组件在此时间。**||
